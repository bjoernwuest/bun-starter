# src/types/ Changes

## Overview

The `src/types/` directory underwent a fundamental restructuring. The previous architecture used `drizzle-orm`'s `InferSelectModel`/`InferInsertModel` to derive TypeScript types directly from Drizzle schema definitions, with `drizzle-typebox`'s `createSelectSchema`/`createInsertSchema` for runtime validation schemas. The new architecture introduces a clear separation between **auto-generated schema types** (`_`-prefixed files, generated by [`scripts/generate_types.ts`](scripts/generate_types.ts)) and **manual user definitions** (non-prefixed files that re-export the generated types and add custom extensions).

Key transformations:

1. **All Drizzle-inferred types removed** from `src/types/`. Types are now derived from TypeBox schemas using `Static<typeof Schema>`.
2. **All `drizzle-orm` and `drizzle-typebox` imports removed** from `src/types/` files.
3. **`src/_types/` directory deleted** — these previously held hand-written types that are now replaced by the auto-generated `_`-prefixed files inside `src/types/`.
4. **`src/types/Database.ts` and `src/types/schema.ts` deleted** — no longer needed.
5. **Server-sent events types** moved from `src/services/` into a dedicated `src/types/ServerSentEvents.ts`.
6. **Naming convention introduced**: `_<Name>.ts` for auto-generated, `<Name>.ts` for manual extensions — enforced by [`src/types/AGENTS.md`](src/types/AGENTS.md).

## New Files

### src/types/AGENTS.md
- **Status**: New (staged, `AM`)
- **Purpose**: AI agent guidance for the types folder
- **Key rules**:
  - `_<name>.ts` files are auto-generated and read-only
  - `<name>.ts` files re-export everything from `_<name>.ts` and add manual extensions
  - Only imports from `@sinclair/typebox`, `elysia`, `react`, and `/src/types/**` are allowed
  - All files must be 100% browser-compatible (no Node.js/backend APIs)

### src/types/ServerSentEvents.ts
- **Status**: New (staged, `AM`)
- **Key exports**:
  - `ServerSentEventEnvelope` interface — `{ topic, data, receivedAt }`
  - `ServerSentEventClientConfig` interface — `{ topics?: readonly string[] }`
  - `ServerSentEventClientSnapshot` interface — `{ sessionKey, topics, createdAt, lastSeenAt, streaming }`
  - `DEFAULT_MAX_BUFFERED_EVENTS` (100), `HEARTBEAT_INTERVAL_MS` (25s), `STALE_TTL_MS` (30min)
  - TypeBox schemas: `SseStreamQuerySchema`, `SseTopicsUpdateBodySchema`, `SseTopicFilterStateSchema`, `SseKnownTopicsResponseSchema`

### src/types/Config.ts
- **Status**: New (untracked)
- **Purpose**: Manual type extensions for Config (replaces deleted `ConfigEntry.ts`)
- Re-exports from `./_Config`

### src/types/AuditEntry.ts
- **Status**: New (untracked)
- **Purpose**: Manual type extensions for AuditEntry
- Re-exports from `./_AuditEntry`

### src/types/_ApiKey.ts, _AuditEntry.ts, _Config.ts, _FunctionalPermission.ts, _User.ts
- **Status**: New (untracked) — auto-generated by `scripts/generate_types.ts`
- **Purpose**: TypeBox schemas generated from Drizzle schema definitions in `src/schema/`
- Each contains `selectSchema`, `insertSchema`, and derived `Type`/`Static` exports
- **Must never be manually edited** per AGENTS.md

## Modified Files

### src/types/AdminApi.ts → src/types/Api.ts (Renamed)
- **Operation**: Rename with modifications (58% similarity)
- **Before**: `AdminApi.ts` — admin-specific API response types using `drizzle-typebox`'s `createSelectSchema` to generate schemas from Drizzle models
- **After**: `Api.ts` — general API shared types using raw TypeBox (both `@sinclair/typebox` and `elysia`'s `t`)
- **Key changes**:
  - Removed imports from `@/schema/User.ts`, `@/schema/FunctionalPermission.ts`, and `drizzle-typebox`
  - Now imports `UserSchema`/`GroupSchema` from `@/types/User.ts` and `FunctionalPermissionSchema` from `@/types/FunctionalPermission.ts`
  - Renamed `EdenUserSchema` → `UserSchema`, `EdenGroupSchema` → `GroupSchema`
  - **Added shared utility schemas**:
    - `SuccessResponseSchema` — `{ success: Boolean }`
    - `OptimisticLockBodySchema` — `{ knownUpdatedAt: String }`
    - `ErrorResponseSchema` — `{ error: String }` (minimal, no message)
    - `HealthResponseSchema` — `{ status, ts }`
    - `FunctionalPermissionsListSchema` — array of `FunctionalPermissionSchema`

### src/types/ApiKey.ts
- **Before**: 9 lines — simple Drizzle-inferred type aliases (`ApiKeyType`, `NewApiKeyType`, `ApiKeyFunctionalPermissionType`, `NewApiKeyFunctionalPermissionType`)
- **After**: 118 lines — comprehensive TypeBox schemas, PubSub topic constants, and typed request/response types
- **Key changes**:
  - Added `export * from './_ApiKey'` to re-export auto-generated types
  - Added PubSub topic constants: `pubsub_ApiKeys`, `pubsub_ApiKeyPermissionsChanged`, `pubsub_ApiKeyCreated`, `pubsub_ApiKeyUpdated`, `pubsub_ApiKeyDisabled`, `pubsub_ApiKeyDeleted`
  - Added `ApiKeySummarySchema` + `ApiKeySummary` type (full summary with permission names)
  - Added `ApiKeysResponseSchema` + `ApiKeysResponse` type (paginated list response)
  - Added `ApiKeyDetailSchema` + `ApiKeyDetailResponse` type (detail with all permissions)
  - Added `CreateApiKeyRequest`/`CreateApiKeyResponse` raw types
  - Added TypeBox operation schemas: `ApiKeyCreateBodySchema`, `ApiKeyCreatedResponseSchema`, `ApiKeyUpdateMetadataBodySchema`, `ApiKeyUpdatedAtResponseSchema`, `ApiKeyProlongBodySchema`, `ApiKeyProlongResponseSchema`, `ApiKeyDisableResponseSchema`, `ApiKeyPermissionsBodySchema`

### src/types/Auth.ts
- **Before**: 8 lines — simple `Claims` record type and `AuthorizedContext` interface
- **After**: 45 lines — expanded with Session interface and Me endpoint TypeBox schemas
- **Key changes**:
  - Changed `Claims` from `Record<string, unknown>` to `Record<string, any>`
  - Changed `FunctionalPermissionType` import to `FunctionalPermission as FunctionalPermissionType` from `./FunctionalPermission.ts`
  - **Added `Session` interface**: `{ idTokenRaw?, idTokenClaims, refreshToken?, accessToken?, expiresAt }`
  - **Added TypeBox schemas**: `MeUserSchema`, `MeContextResponseSchema` (with `permissionNames` and `functionalPermissions`)

### src/types/FunctionalPermission.ts

- **After**: 19 lines — clean TypeBox body schemas

## Architecture Summary

### Before (old pattern)
```
src/types/
├── AdminApi.ts        ← drizzle-typebox createSelectSchema()
├── ApiKey.ts          ← drizzle-orm InferSelectModel/InferInsertModel
├── Auth.ts            ← typed interfaces
├── ConfigEntry.ts     ← drizzle-typebox + drizzle-orm
├── Database.ts        ← DB connection types
├── FunctionalPermission.ts ← drizzle-orm inferred types
├── RequestBundling.ts ← plain TS interfaces
├── schema.ts          ← re-exports
├── User.ts            ← drizzle-orm inferred types
├── Config.type.ts     ← interim refactor file
├── User.type.ts       ← interim refactor file
└── API.ts             ← capital acronym variant

src/_types/            ← hand-written types (deleted)
├── AdminApi.ts
├── ApiKey.ts
├── Auth.ts
├── ConfigEntry.ts
├── FunctionalPermission.ts
├── Page.ts
├── User.ts
└── helpers.ts
```

### After (new pattern)
```
src/types/
├── AGENTS.md                      ← AI guidance (NEW)
├── _ApiKey.ts                     ← auto-generated TypeBox schema (NEW)
├── _AuditEntry.ts                 ← auto-generated TypeBox schema (NEW)
├── _Config.ts                     ← auto-generated TypeBox schema (NEW)
├── _FunctionalPermission.ts       ← auto-generated TypeBox schema (NEW)
├── _User.ts                       ← auto-generated TypeBox schema (NEW)
├── ApiKey.ts                      ← manual: re-exports _ApiKey + PubSub topics + operation schemas
├── Api.ts                         ← manual: general API shared schemas (renamed from AdminApi.ts)
├── AuditEntry.ts                  ← manual: re-exports _AuditEntry (NEW)
├── Auth.ts                        ← manual: Session + Me endpoint schemas
├── Config.ts                      ← manual: re-exports _Config (NEW, replaces ConfigEntry.ts)
├── FunctionalPermission.ts        ← manual: re-exports _FunctionalPermission + body schemas
├── helpers.ts                     ← (unchanged, helper utilities)
├── Page.ts                        ← (unchanged)
├── RequestBundling.ts             ← manual: + fallback constants + TypeBox schema
├── ServerSentEvents.ts            ← NEW: SSE types and TypeBox schemas
└── User.ts                        ← manual: re-exports _User (minimal)
```
